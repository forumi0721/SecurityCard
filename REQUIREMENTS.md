# 보안카드 관리 Android 앱 요건 정의서

> **버전**: 2.0.0-draft
> **작성일**: 2026-06-16
> **기반**: 레거시 SecurityCard v1.0.12 소스 분석 및 현대화 재개발

---

## 1. 개요

본 앱은 사용자의 보안카드 정보를 안전하게 저장하고, 필요 시 빠르게 조회할 수 있는 Android 전용 보안카드 관리 앱이다.

레거시(v1.0.12) 분석을 통해 확인된 핵심 기능을 유지하면서, 현대적인 Android 개발 스택과 UX 패턴으로 완전히 재개발한다.

### 레거시 분석 요약

레거시 앱(Android minSdk 9, Java, XML 레이아웃)에서 확인된 주요 기능:

- 보안카드 단일 기능으로 구성 (기존 은행통장 기능은 사용자 요청에 따라 완전히 제거)
- 보안카드 코드 입력: 최대 35개 셀, 3자리 또는 4자리 선택
- 코드 조회: 번호(index) 방식 조회 (첫 번째, 두 번째 입력 → 코드 출력)
- 보안카드 역조회: 앞/뒤 두 자리로 위치 검색
- 전체 코드 보기(Grid 형태) + 코드 검색 보기 두 가지 모드
- 비밀번호 인증 (숫자 입력)
- 입력 오류 지연(1~5초), 자동 로그아웃(10~60분) 설정
- 정렬: 이름순 / 등록순 / 조회순
- 알림바 아이콘 표시 (빠른 앱 복귀용)
- DB 백업/복구 (외부 저장소)
- AES/DES 암호화 (레거시는 ECB 모드, 재개발 시 GCM으로 업그레이드)
- 콜센터 전화번호 저장 및 연결
- 보안카드 Export/Import (파일 기반)
- 코드 입력 시 "연속입력" 토글 기능

## 3. 주요 기능

### 3.1 최초 실행 및 인증 등록

앱을 처음 실행하면 인증 수단을 등록한다.

**필수 등록:**
- PIN (최소 4자리 ~ 최대 8자리 숫자)

**선택 등록:**
- 생체 인증 (지문 / 안면) — 단말 지원 시에만 활성화

**PIN 정책:**
- 원문 저장 금지 — PBKDF2WithHmacSHA256으로 해시 후 저장
- Salt는 Android Keystore에서 생성/보관
- 연속 입력 실패 시 지연 적용 (설정 가능: 사용안함 / 1~5초)

### 3.2 로그인

앱 실행 시 로그인 화면을 표시한다.

**로그인 방식:**
- PIN 입력 (숫자 키패드 UI)
- 생체 인증 (BiometricPrompt)

**인증 성공:** 메인 화면으로 이동
**인증 실패:** 오류 메시지 + 지연 시간 적용
**자동 로그아웃:** 설정한 시간(10분~60분) 동안 미사용 시 자동 로그아웃

### 4.4 메인 화면

#### 탭 구성 (레거시 참조)

레거시는 보안카드 탭과 은행통장 탭 두 가지를 지원했으나, 이번 리뉴얼에서는 **보안카드 관리 전용**으로 구성하며 은행통장 기능은 완전히 제거한다.

#### 보안카드 목록 표시 항목

| 항목 | 설명 |
|------|------|
| 보안카드 이름 | 카드 이름 |
| 보안카드 번호 | 카드 고유번호 |
| 콜센터 연락처 | 설정된 경우 표시 |
| 코드 개수 | 등록된 코드 셀 수 |
| 최근 조회일 | 마지막으로 조회한 날짜 |
| 최근 수정일 | 마지막 수정 날짜 |

#### 정렬 기준 (레거시: 이름순/등록순/조회순)

- 이름순 (기본)
- 등록일순
- 수정일순
- **조회순** (많이 조회한 순 — 레거시에서 실제 조회 횟수 관리 확인됨)

#### 검색

- 상단 검색 바: 보안카드 이름 / 카드 번호 검색
- 실시간 필터링

#### 동작

| 액션 | 결과 |
|------|------|
| 카드 클릭 | 보안카드 조회 화면 이동 |
| 길게 누르기 | 다중 선택 모드 진입 |
| 다중 선택 + 삭제 버튼 | 선택 항목 삭제 확인 다이얼로그 → 삭제 |
| FAB 버튼 | 보안카드 등록 화면 이동 |
| 설정 아이콘 | 설정 화면 이동 |

#### 빈 목록 상태

- 빈 목록 안내 메시지 표시 (레거시: "등록된 보안카드가 없습니다. 보안카드를 등록 후 사용하세요.")
- 등록 유도 버튼 표시

### 4.5 보안카드 등록/수정 화면

#### 입력 항목

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 보안카드 이름 | 카드 식별 이름 | 필수 |
| 보안카드 번호 | 카드 고유번호 (숫자) | 필수 |
| 콜센터 연락처 | 전화번호 (전화 연결 가능) | 선택 |
| 코드 자릿수 | 3자리 / 4자리 선택 (레거시 동일) | 필수 |
| 카드 가로 셀 개수 (columnCount) | 그리드 열 수 | 필수 |
| 카드 세로 셀 개수 (rowCount) | 그리드 행 수 | 필수 |
| 카드 방향 | 가로형 / 세로형 | 필수 |
| 각 셀의 보안코드 값 | 코드 자릿수에 맞는 숫자 | 필수 |

#### 코드 입력 방식 (레거시 참조)

- 하단에 실제 보안카드 형태의 Grid UI 표시
- 각 셀을 클릭하면 코드 입력 다이얼로그 표시
  - 셀 번호(No.) 표시 (레거시: "No. %02d" 포맷)
  - 숫자 입력 필드 (maxLength: 코드 자릿수, 최대 4)
  - **연속 입력 토글**: ON 상태에서 확인 시 자동으로 다음 셀 입력 다이얼로그 표시 (레거시 기능)
  - 확인 / 취소 버튼
- 코드가 입력된 셀은 시각적으로 구분 (채워진 표시)
- 코드 자릿수 변경 시 이미 입력된 코드가 있으면 경고 후 초기화 확인 (레거시: "코드가 입력된 상태에서는 자릿수를 변경할 수 없습니다.")
- 최대 셀 개수: 35개 (레거시 CODE_MAX = 35)

#### 저장 조건

- 보안카드 이름 필수
- 보안카드 번호 필수
- 코드 자릿수 설정 필수
- 가로/세로 셀 수 각각 1 이상
- 각 셀 값은 설정된 자릿수의 숫자
- 빈 셀(미입력 셀) 포함 저장 허용
- 저장 시 전체 데이터를 AES-256-GCM으로 암호화

#### 수정 모드

- 기존 보안카드 데이터를 불러와 동일 화면에서 수정
- 뒤로 가기 시 변경사항이 있으면 확인 다이얼로그 표시 (레거시: "수정된 내용을 적용하지 않고 돌아갑니다.")

#### Export / Import (레거시: 화면 상단 메뉴)

- 등록/수정 화면 상단 메뉴에서 Export / Import 접근 가능
- 또는 설정 화면에서도 접근 가능 (중복 제공)

### 4.6 보안카드 조회 화면

조회 방식은 탭으로 제공한다 (레거시의 SecurityViewAll + SecurityViewCode 통합).

#### 공통 영역

- 상단: 보안카드 이름 + 카드 번호
- 이전/다음 카드 이동 버튼 (레거시: 화살표 버튼으로 카드 간 이동, 경계에서 토스트 안내)
- 카드 삭제 / 수정 메뉴 (상단 더보기 메뉴)
- 콜센터 연결 버튼 (저장된 경우)

---

#### 방식 1. 전체 코드 보기 (레거시: SecurityViewAll)

- 보안카드 전체 코드를 Grid 형태로 표시
- 셀 번호(No.)와 코드값을 함께 표시
- 각 셀 클릭 시 표시 방식 전환:
  - 마스킹(`****`) → 코드 전체 표시 → 마스킹 (토글 순환)
- 보안 경고 문구 표시: "보안카드 코드표가 타인에게 노출되지 않도록 관리에 주의하시기 바랍니다."

---

#### 방식 2. 코드 번호 조회 (레거시: SecurityViewCode)

사용자가 코드 번호(index)를 입력하면 해당 위치의 보안카드 값을 출력한다.

**입력 항목:**
- 첫 번째 코드 번호 (1 ~ 코드 개수, 최대 2자리)
- 두 번째 코드 번호 (1 ~ 코드 개수, 최대 2자리)

**자동 진행 (레거시 동일):**
- 첫 번째 번호가 유효한 범위이면 두 번째 입력으로 자동 포커스 이동

**출력 항목:**
- 첫 번째 위치의 코드값 (마스킹/표시 토글 클릭 가능)
- 두 번째 위치의 코드값 (마스킹/표시 토글 클릭 가능)
- 각 코드값 복사 버튼 (레거시: 클립보드 복사)

**초기화 버튼 (레거시: refresh/revert 아이콘):**
- 입력값 및 출력값 초기화

**조회 결과 알림 (레거시 기능 — 재개발 시 보안 강화):**
- 레거시: 상단 알림바에 검색 결과 표시 후 클릭 시 폐기 요청
- 재개발: 인앱 표시 우선, 알림 발송 옵션은 설정에서 제어

---

#### 방식 3. 보안카드 값 역조회 (레거시 기능 유지)

사용자가 앞 두 자리 또는 뒤 두 자리를 입력하면 해당 값이 포함된 코드 위치를 조회한다.

**입력 항목:**
- 앞 두 자리 (선택)
- 뒤 두 자리 (선택)

**출력 항목:**
- 일치하는 셀 번호 목록
- 해당 셀의 전체 코드값

**예시:**
- 앞 두 자리: `12`, 뒤 두 자리: `89` → 결과: `No.03 [1289]`, `No.17 [1289]` 등

## 5. Export / Import

### Export

- 현재 저장된 보안카드(및 은행통장) 데이터를 파일로 내보낸다.
- Export 파일은 AES-256-GCM 암호화된 JSON 포맷으로 저장한다.
- Export 시 사용자가 백업 비밀번호를 입력한다 (PBKDF2 기반 키 유도).
- 저장 경로: Android 공개 Documents 또는 Downloads 디렉토리 (SAF 사용).
- **레거시 주의**: 레거시는 암호화 없이 외부 저장소에 저장했음 → 재개발 시 반드시 암호화 적용.

### Import

- Export 파일을 선택하여 데이터를 복원한다.
- Import 시 백업 비밀번호를 입력한다.
- 복원 전 "기존 데이터 유지(병합) / 덮어쓰기" 여부를 선택할 수 있다.
- 파일 형식 오류 또는 비밀번호 오류 시 안내 메시지 표시.
- **레거시 기능**: Import 파일 목록 표시 및 파일 삭제 기능 → 재개발에서도 유지.

## 6. 공통 요건

### 6.1 실행 중 알림 (Notification)

- 앱 로그인 후 상단 알림바에 아이콘/텍스트를 표시한다. (레거시 기능)
- 알림 클릭 시 앱의 현재 화면 또는 메인 화면으로 이동.
- 앱 종료 또는 로그아웃 시 알림 제거.
- 재개발: Foreground Service 없이 Notification 채널로 구현.
- 알림 표시 여부는 설정에서 ON/OFF 가능.

### 6.2 보안 요건

| 요건 | 설명 |
|------|------|
| 데이터 암호화 | Room DB 저장 전 AES-256-GCM 암호화 |
| Keystore 사용 | 암호화 키는 Android Keystore에서 생성/관리 |
| PIN 저장 | PBKDF2WithHmacSHA256 + Salt로 해시 후 저장 |
| 스크린샷 방지 | WindowManager.LayoutParams.FLAG_SECURE 적용 |
| 최근 앱 화면 보호 | 최근 앱 목록에서 내용 노출 방지 |
| 앱 재진입 재인증 | 설정된 시간 초과 후 재진입 시 로그인 화면으로 |
| 자동 로그아웃 | 설정된 시간 동안 미사용 시 자동 로그아웃 |
| 입력 실패 지연 | PIN 오류 시 설정된 시간만큼 재입력 대기 |
| 평문 저장 금지 | 모든 민감 데이터는 암호화 후 저장 |

### 6.3 접근성

- 모든 인터랙티브 요소에 contentDescription 제공
- 시스템 폰트 크기 변경 대응 (sp 단위 사용)
- 명도 대비 WCAG AA 수준 준수

## 8. 화면 구성

```
앱 실행
  └─ Splash 화면
       ├─ 최초 실행 → 최초 인증 등록 화면
       └─ 재실행 → 로그인 화면
                     └─ 인증 성공
                           └─ 메인 화면 (보안카드 목록)
                               ├─ 보안카드 등록/수정 화면
                               │    └─ 코드 입력 다이얼로그 (연속입력 토글)
                               ├─ 보안카드 조회 화면
                               │    ├─ 전체 코드 보기 탭 (마스킹 토글)
                               │    ├─ 코드 번호 조회 탭 (첫번째/두번째 입력)
                               │    └─ 역조회 탭 (앞/뒤 두자리 검색)
                               └─ 설정 화면
                                    └─ PIN 변경 화면
```

## 10. 예외 처리

| 상황 | 처리 방법 |
|------|----------|
| PIN 인증 실패 | 오류 메시지 + 설정된 지연 시간 적용 |
| 생체 인증 미지원 | 생체 인증 옵션 비활성화 |
| 생체 인증 미등록 | 안내 메시지 → 시스템 설정 유도 |
| 보안카드 셀 미입력 | 저장 시 경고 메시지 표시 |
| 코드 자릿수 변경 시 기입력 데이터 존재 | 경고 후 초기화 확인 다이얼로그 |
| Import 파일 형식 오류 | "보안카드 파일이 아닙니다." 안내 |
| Import 비밀번호 오류 | "비밀번호가 일치하지 않습니다." 안내 |
| DB 복호화 실패 | "데이터 손상" 안내 + 복구 옵션 제시 |
| 조회 범위 초과 | "보안코드 범위에 맞게 입력해 주십시오." 안내 |
| 코드 없음 | "등록된 코드가 없습니다." 안내 |
| 콜센터 미등록 | "콜센터 전화번호를 설정해 주십시오." 안내 |
| 백업 폴더 생성 실패 | 오류 안내 Toast |
| 네트워크 오류 | 현재 버전은 온라인 기능 없음 (완전 오프라인) |

## 11. 완료 기준

- [ ] 사용자는 최초 실행 시 PIN을 등록할 수 있다.
- [ ] 사용자는 PIN 또는 생체 인증으로 로그인할 수 있다.
- [ ] 사용자는 보안카드를 등록, 조회, 수정, 삭제할 수 있다.
- [ ] 코드 입력 시 연속 입력 모드로 빠르게 입력할 수 있다.
- [ ] 사용자는 코드 번호(index) 방식으로 코드값을 조회할 수 있다.
- [ ] 사용자는 앞/뒤 두 자리로 역조회를 할 수 있다.
- [ ] 전체 코드 보기에서 셀 클릭 시 마스킹/표시를 전환할 수 있다.
- [ ] 이전/다음 카드 이동이 조회 화면에서 가능하다.
- [ ] 정렬(이름순/등록일순/수정일순/조회순) 및 검색이 동작한다.
- [ ] 사용자는 데이터를 암호화하여 Export / Import 할 수 있다.
- [ ] 앱 실행 중 알림을 통해 빠르게 앱으로 복귀할 수 있다.
- [ ] 저장 데이터는 평문으로 노출되지 않는다.
- [ ] 스크린샷 방지 및 최근 앱 화면 보호가 적용된다.
- [ ] 자동 로그아웃 및 재진입 시 재인증이 동작한다.

## 12. 레거시 → 재개발 주요 변경 사항

| 항목 | 레거시 (v1.0.12) | 재개발 (v2.0) |
|------|-----------------|--------------|
| 언어 | Java | Kotlin |
| UI | XML 레이아웃 + Activity | Jetpack Compose + Navigation |
| 인증 | 숫자 비밀번호 (SharedPreferences 저장) | PIN 해시 (PBKDF2) + BiometricPrompt |
| 암호화 | AES-128-ECB (고정 키 하드코딩) | AES-256-GCM + Android Keystore |
| 저장 | SQLite + DBAdapter 직접 구현 | Room Database + Repository 패턴 |
| Export | 암호화 없이 외부 저장소 저장 | PBKDF2 기반 암호화 JSON |
| 광고 | Google AdMob 광고 포함 | 광고 제거 |
| 아키텍처 | Activity 중심 God Object | MVVM + Clean Architecture |
| 최소 SDK | API 9 (Android 2.3) | API 26 (Android 8.0) |
| 네트워크 권한 | INTERNET, ACCESS_NETWORK_STATE | 불필요 (완전 오프라인) |
| 외부 저장소 | WRITE_EXTERNAL_STORAGE | SAF (Storage Access Framework) |
